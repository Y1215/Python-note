Flume的开发案例

1.实时读取本地文件到HDFS

实时监控Hive日志，并上传到HDFS

1.1 导入Hadoop相关jar包

[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/common/lib/commons-configuration-1.6.jar /usr/local/flume-1.7.0/lib/
[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/common/lib/commons-io-2.4.jar /usr/local/flume-1.7.0/lib/
[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/common/lib/hadoop-auth-2.8.1.jar /usr/local/flume-1.7.0/lib/
[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/common/lib/htrace-core4-4.0.1-incubating.jar /usr/local/flume-1.7.0/lib/
[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/common/hadoop-common-2.8.1.jar /usr/local/flume-1.7.0/lib/
[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/hdfs/hadoop-hdfs-2.8.1.jar /usr/local/flume-1.7.0/lib/

1.2 创建flume-file-hdfs.conf

[root@master conf]# vi /usr/local/flume-1.7.0/conf/flume-file-hdfs.conf

1.3 执行监控配置

[root@master ~]# flume-ng agent --conf conf/ --name a2 --conf-file flume-file-hdfs.conf

1.4 开启hive并操作hive

[root@master ~]# hive
hive>

1.5 在hdfs上查看

http://master:50070/

2.实时读取目录文件到HDFS

2.1 创建flume-dir-hdfs.conf

[root@master conf]# vi /usr/local/flume-1.7.0/conf/flume-dir-hdfs.conf

2.2 执行监控配置

[root@master conf]# ../bin/flume-ng agent --conf conf/ --name a3 --conf-file flume-dir-hdfs.conf

  • 说明:在使用Spooling Directory Source时
    • 不要在监控目录中创建并持续修改文件
    • 上传完成的文件会以.COMPLETED结尾
    • 被监控文件夹每500毫秒扫描一次文件变动

2.3 向upload文件夹中添加文件

[root@master ~]# cd /usr/local/flume-1.7.0/upload/
[root@master upload]# touch test.txt
[root@master upload]# touch test.tmp
[root@master upload]# touch test.log

2.4 在hdfs上查看

http://master:50070/
