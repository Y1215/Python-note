Flume的开发案例

1.实时读取本地文件到HDFS

实时监控Hive日志，并上传到HDFS

1.1 导入Hadoop相关jar包

[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/common/lib/commons-configuration-1.6.jar /usr/local/flume-1.7.0/lib/
[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/common/lib/commons-io-2.4.jar /usr/local/flume-1.7.0/lib/
[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/common/lib/hadoop-auth-2.8.1.jar /usr/local/flume-1.7.0/lib/
[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/common/lib/htrace-core4-4.0.1-incubating.jar /usr/local/flume-1.7.0/lib/
[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/common/hadoop-common-2.8.1.jar /usr/local/flume-1.7.0/lib/
[root@master conf]# cp /usr/local/hadoop-2.8.1/share/hadoop/hdfs/hadoop-hdfs-2.8.1.jar /usr/local/flume-1.7.0/lib/

1.2 创建flume-file-hdfs.conf

[root@master conf]# vi /usr/local/flume-1.7.0/conf/flume-file-hdfs.conf

1.3 执行监控配置

[root@master ~]# flume-ng agent --conf conf/ --name a2 --conf-file flume-file-hdfs.conf

1.4 开启hive并操作hive

[root@master ~]# hive
hive>

1.5 在hdfs上查看

http://master:50070/

2.实时读取目录文件到HDFS

2.1 创建flume-dir-hdfs.conf

[root@master conf]# vi /usr/local/flume-1.7.0/conf/flume-dir-hdfs.conf

2.2 执行监控配置

[root@master conf]# ../bin/flume-ng agent --conf conf/ --name a3 --conf-file flume-dir-hdfs.conf

  • 说明:在使用Spooling Directory Source时
    • 不要在监控目录中创建并持续修改文件
    • 上传完成的文件会以.COMPLETED结尾
    • 被监控文件夹每500毫秒扫描一次文件变动

2.3 向upload文件夹中添加文件

[root@master ~]# cd /usr/local/flume-1.7.0/upload/
[root@master upload]# touch test.txt
[root@master upload]# touch test.tmp
[root@master upload]# touch test.log

2.4 在hdfs上查看

http://master:50070/

2.5 查看upload文件夹

3.高可用Flum-NG配置

3.1 角色分配

Flume的Agent和Collector分布如下所示：

名称 HOST 角色
Agent slave1 Web Server
Collector1 master AgentMstr1
Collector2 slave3 AgentMstr2

3.2 配置

3.2.1 拷贝Flume

[root@master local]# scp -r flume-1.7.0/ slave1:$PWD
[root@master local]# scp -r flume-1.7.0/ slave2:$PWD

3.2.2 agent配置

[root@slave1 flume-1.7.0]# vi conf/agent.conf

3.2.3 collector配置

[root@master ~]# cd /usr/local/flume-1.7.0/conf/
[root@master conf]# vim collector.conf

[root@slave3 ~]# cd /usr/local/flume-1.7.0/conf/
[root@slave3 conf]# vim collector.conf

3.4 启动

  • master上启动flume

    [root@master ~]# cd /usr/local/flume-1.7.0/conf/
    [root@master conf]# ../bin/flume-ng agent -n a1 -c conf -f collector.conf -Dflume.root.logger=DEBUG,console

  • slave3上启动flume

    [root@slave3 ~]# cd /usr/local/flume-1.7.0/conf/
    [root@slave3 conf]# ../bin/flume-ng agent -n a1 -c conf -f collector.conf -Dflume.root.logger=DEBUG,console

  • slave1上启动flume

    [root@slave1 ~]# cd /usr/local/flume-1.7.0/conf/
    [root@slave1 conf]# ../bin/flume-ng agent -n a1 -c conf -f agent.conf -Dflume.root.logger=DEBUG,console

  • slave1上启动脚本

    [root@slave1 ~]# cd /usr
    [root@slave1 usr]# mkdir /shells
    [root@slave1 usr]# mkdir /shells/logs
    [root@slave1 usr]# cd shells
    [root@slave1 shells]# vi tail-file.sh

    • [root@slave1 shells]# sh tail-file.sh

4.两个agent级联

需求分析：第一个agent负责收集文件当中的数据，通过网络发送到第二个agent当中去，第二个agent负责接收第一个agent发送的数据，并将数据保存到hdfs上面去

4.1 slave1配置flume配置文件

[root@slave1 ~]# cd /usr/local/flume-1.7.0/conf/
[root@slave1 conf]# vi tail-avro.conf

4.2 在master上配置文件

[root@master ~]# cd /usr/local/flume-1.7.0/conf/
[root@master conf]# vi avro-hdfs.conf

4.3 顺序启动

  • master启动

    [root@master conf]# ../bin/flume-ng agent -c conf -f avro-hdfs.conf -n a1 -Dflume.root.logger=INFO,console

  • slave1启动

    [root@slave1 conf]# ../bin/flume-ng agent -c conf -f tail-avro.conf -n a1 -Dflume.root.logger=INFO,console

  • slave1运行脚本

    [root@slave1 ~]# cd /usr/shells/
    [root@slave1 shells]# sh tail-file.sh h
